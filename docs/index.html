How it works

Print one thing. Leave everything else alone.

The component builds an isolated print document in an invisible iframe — the host page is never touched.

Isolated, Not Invasive

The host page DOM is never mutated. No flicker, no scroll jump, no class toggling. Print prep happens entirely inside a 1×1 px off-screen iframe that is removed the moment the dialog closes.

Shadow DOM Ready

Uses getHTML({ serializableShadowRoots: true }) to capture rendered shadow DOM as declarative HTML. Components built with Stapler print faithfully without any JavaScript re-execution in the print frame.

Predictable Lifecycle

Every print-start event is guaranteed a matching print-end. Cancel before the dialog and nothing fires. Cancel after print-start — print-end still fires. A 60-second fallback guards against browsers that never fire afterprint.

<print-element-button>

Print any DOM element. Zero dependencies.

A single-file custom element that captures one element and its descendants — including shadow DOM via getHTML() — into an isolated iframe, then triggers the browser's native print dialog. The host page DOM is never mutated: no flicker, no scroll jump, no class toggling.

✓ No dependencies
✓ Shadow DOM support
✓ Zero DOM mutation
✓ Works with Stapler
✓ Promise-based API
✓ Consistent lifecycle
MIT License · Version 1.0.0 · github.com/rlnorthcutt/print-element-button

Quick Start

Include the script as a module — it registers the custom element automatically on import.

<script type="module" src="print-element-button.js"></script>

<div id="my-document">
  <!-- the content you want to print -->
</div>

<print-element-button target="#my-document"
                      page-size="letter"
                      margins="0.75in"
                      print-title="My Report">
  Print Document
</print-element-button>

The button label is slotted — anything between the tags becomes the label. Falls back to "Print" if the element is empty.

Attributes

Attribute Default Description
target parentElement CSS selector for the element to print. Resolved at click time. When a selector matches multiple elements, the first match in document order is used. If omitted, uses the component's parent element.
page-size letter Passed to @page { size }. Accepts letter, legal, a4, or explicit dimensions such as 8.5in 11in.
margins 0 Passed to @page { margin }. Accepts any CSS length value or shorthand, e.g. 0.75in or 1in 0.5in.
print-title page title Title shown in the print dialog and used as the default filename when saving as PDF. Inherits the current page title if omitted.

Methods

print() → Promise<void> — Triggers the full print sequence. Resolves after the dialog closes (afterprint). Rejects with DOMException('InvalidStateError') if already printing, or with an Error if target resolution or iframe setup fails.

cancel() — Aborts an in-flight print before the dialog opens. If called after print-start has already fired, print-end still fires — the lifecycle is always balanced.

Events

All events bubble and are composed. Every print-start is guaranteed a matching print-end.

Event Fires when detail
print-start iframe is ready and print() has been called on it null
print-end afterprint fires, the 60 s timeout expires, or cancel() is called after start null; { timedOut: true } on timeout
print-error setup fails or print() is blocked by the browser { error: Error }

Browser Support

Browser Minimum version Shadow DOM serialization
Chrome 125+  (May 2024) ✓ Full — via getHTML()
Firefox 128+  (July 2024) ✓ Full — via getHTML()
Safari 17.4+  (March 2024) Light DOM only — getHTML() not yet shipped

Known Limitations

  • Shadow DOM capture requires getHTML() — Chrome & Firefox only. On Safari, shadow root internals are not included in the print output.
  • Form state, scroll position, and canvas content do not serialize via outerHTML or getHTML().
  • Closed shadow roots are not captured — use open shadow DOM.
  • Viewport units (vh, vw) resolve against the invisible 1 px iframe. Use absolute units (in, cm) for page-dimension CSS.
  • Images with loading="lazy" are rewritten to eager automatically, but images not yet fetched in the host page may still be absent from the print output.
